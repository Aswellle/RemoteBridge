# RemoteBridge 使用说明书

> 远程文件桥接系统 —— 服务启动、桌面端运行与全功能操作指南
> 适用版本:v1.0.0(2026-06 验证通过)

---

## 一、系统组成

RemoteBridge 由三部分组成,按以下关系协作:

```
┌──────────────┐   出站 WebSocket   ┌──────────────┐   WebSocket   ┌──────────────┐
│  桌面端 Host  │ ─────────────────→ │  Relay 中继   │ ←──────────── │  Web 客户端   │
│ (Electron)   │                    │  服务器       │               │  (浏览器)     │
│ 你的电脑      │                    │  :3001       │               │  :3000       │
└──────────────┘                    └──────────────┘               └──────────────┘
```

- **Relay 中继服务器**:消息转发枢纽,不存储任何文件。默认端口 `3001`。
- **桌面端(Host)**:安装在被访问电脑上的 Electron 应用,管理共享目录、生成连接码、审计访问。**永不对外开放端口**,只主动连接 Relay。
- **Web 客户端**:浏览器访问,凭连接码(PIN)连接到桌面端,远程浏览/预览/下载文件、收发消息。默认端口 `3000`。

**启动顺序建议:Relay → 桌面端 → Web**(顺序错了也能自愈,桌面端与 Web 都有无限重连机制)。

## 二、环境要求

| 依赖 | 版本 | 说明 |
|---|---|---|
| Node.js | 22.x | 项目实测于 v22.14.0 |
| pnpm | 9.x | `npm i -g pnpm` 或 corepack |
| 操作系统 | Windows / macOS / Linux | 本文以 Windows 为例 |

## 三、首次安装

在项目根目录(`D:\AI\remotebridge`)执行:

```sh
# 1. 安装全部依赖
pnpm install

# 2. 构建共享协议包(必须先于一切应用构建/运行)
pnpm --filter @remotebridge/shared build
```

> ⚠️ **better-sqlite3 注意事项**:服务器(Node)与桌面端(Electron)需要不同的原生二进制。
> 仓库已内置自动切换机制(`.cache/better_sqlite3.electron.node` + dlopen 钩子)。
> 若 Relay 启动时报 `NODE_MODULE_VERSION` 不匹配错误,执行:
> ```sh
> pnpm --filter @remotebridge/server rebuild better-sqlite3
> ```

## 四、启动服务

### 4.1 启动 Relay 中继服务器

```sh
pnpm --filter @remotebridge/server dev
```

看到以下输出即成功:

```
🚀 RemoteBridge Relay Server 启动于 http://0.0.0.0:3001
📡 WebSocket 端点: ws://0.0.0.0:3001/ws
🔗 健康检查: http://0.0.0.0:3001/health
```

验证:浏览器或 curl 访问 `http://127.0.0.1:3001/health`,返回 `{"status":"ok",...}`。

可选环境变量(生产部署时建议显式设置):

```
RELAY_PORT=3001            # 监听端口
JWT_SECRET=<随机长串>       # access token 签名密钥
JWT_REFRESH_SECRET=<另一串> # refresh token 签名密钥
ALLOWED_ORIGINS=https://your-domain.com # CORS 来源白名单(逗号分隔多个;留空时回退为 http://localhost:3000)
RB_DATA_DIR=<目录>          # 数据库位置,默认 ~/.remotebridge/data
```

生产模式:`pnpm --filter @remotebridge/server build && pnpm --filter @remotebridge/server start`

> **生产部署安全要求**(详见 `.env.example` 与 `docker-compose.yml`):
> - `ALLOWED_ORIGINS` 必须显式设置为 Web 客户端的真实来源(协议+域名,逗号分隔多个)。
>   未设置时回退为 `http://localhost:3000`,会导致部署后的 Web 客户端被 CORS 拦截。
> - Relay 务必部署在 TLS 之后(参见 `docker-compose.yml` 中的 Caddy 反向代理,自动签发
>   Let's Encrypt 证书)。PIN 码、JWT access/refresh token、`register-host` 返回的 Host
>   secret 均经此连接传输,明文 HTTP 会暴露这些凭证。

### 4.2 启动 Web 客户端

```sh
pnpm --filter @remotebridge/web dev
```

启动后浏览器访问 **http://localhost:3000**。

生产模式:`pnpm --filter @remotebridge/web build && pnpm --filter @remotebridge/web start`

### 4.3 构建并打开桌面端程序

**方式 A:构建后直接运行(推荐,日常使用)**

```sh
# 构建(产物在 apps/desktop/dist/)
pnpm --filter @remotebridge/desktop build

# 启动(Windows,在项目根目录执行)
cd apps/desktop
.\node_modules\.bin\electron .
```

**方式 B:开发模式(带热重载,改代码时用)**

```sh
pnpm --filter @remotebridge/desktop dev
```

> 若开发模式因 better-sqlite3 报错,改用 `scripts/dev-desktop.ps1`(会自动 rebuild),
> 之后记得为服务器执行 `pnpm rebuild better-sqlite3` 恢复。

**方式 C:打包为安装程序(分发给最终用户)**

```sh
pnpm --filter @remotebridge/desktop package:win    # Windows(另有 :mac / :linux)
```

产物为 electron-builder 安装包,安装后从开始菜单启动。

**一键启动全部(开发用)**:根目录执行 `pnpm dev`(turbo 并行启动三端)。

## 五、桌面端操作流程

### 5.1 首次启动与自动连接

桌面端启动后会**自动注册并连接 Relay**,窗口左下角显示"已连接"即就绪。主机身份(hostId)持久化保存,重启后复用,不会重复注册。

> ⚠️ 桌面端默认连接 `ws://127.0.0.1:3001/ws`。**不要使用 `localhost`** —— 它会被解析为
> IPv6 `::1` 而 Relay 只监听 IPv4,导致连接失败。如需连接远程 Relay,在「设置」页
> 填写实际 IP 或域名。

配置文件位置(手工排查时使用):
`C:\Users\<用户名>\AppData\Roaming\@remotebridge\desktop\remotebridge-config.json`

### 5.2 添加共享目录(必做)

Web 端只能访问白名单中的目录:

1. 左侧导航进入 **「共享目录」**;
2. 点击添加,选择要共享的文件夹;
3. 为目录设置权限:
   - **只读**:Web 端可浏览、预览;
   - **可下载**:在只读基础上允许下载文件;
4. 可为目录设置别名(Web 端显示的名称)。

> 系统关键目录(如 `C:\Windows`、`C:\Program Files`)被内置黑名单拦截,无法共享。

### 5.3 生成连接码(PIN)

1. 进入 **「连接状态」** 页;
2. 点击 **「生成连接码」**,得到 8 位 PIN(如 `7RKY-N5G2`);
3. 把 PIN 告知要连接的人(或自己在浏览器输入)。

PIN 规则:
- **一次性使用**,连接成功即失效;
- 默认 **5 分钟过期**(可在生成时选择时长);
- 字符集排除易混淆字符(无 0/O/I/1/l);
- 每分钟最多生成 5 次。

### 5.4 已连接客户端管理(信任 / 吊销)

进入 **「已连接客户端」** 页,列表实时显示每个客户端的在线状态:

- **信任 / 取消信任**:为设备打信任标记(本地记录,可随时切换);
- **吊销**:弹出确认后立即作废该会话 —— 对方被**强制断开**并踢回连接页,必须重新获取 PIN 才能连接;
- **活动日志**标签页:查看每个客户端的文件访问记录(含被拦截的越权请求)。

### 5.5 消息中心 / 安全审计 / 设置

- **消息中心**:与已连接的 Web 客户端互发文字消息;
- **安全审计**:查看全部安全事件(连接、吊销、路径拦截等);
- **设置**(全部**保存即生效**,无需重启):
  - Relay 服务器地址(WebSocket + API):变更后自动断开并按新地址重连,结果即时反馈;
  - 开机自启 / 最小化到托盘;
  - 主题(暗色 / 亮色):保存后立即切换。

## 六、Web 端操作流程

### 6.1 连接到远程电脑

1. 浏览器打开 **http://localhost:3000**;
2. 输入桌面端生成的 8 位连接码;
3. 填写设备名称(下次连接自动记住);
4. 点击「连接」→ 进入 dashboard。

会话有效期:访问令牌 2 小时(自动刷新),刷新令牌 30 天。**刷新页面不会丢会话**。

### 6.2 文件浏览 / 预览 / 下载

进入 **「文件浏览」**:

- 首屏显示主机授权的**共享目录列表**,点击进入逐级浏览(面包屑可回跳);
- 点击文件 → 弹出**预览**(文本/图片/PDF,上限 10MB);
- 预览框内或文件行点击 **「下载」** → 右侧「下载管理」面板显示实时进度;
- 下载走 Relay 中继隧道,支持**断点续传**(HTTP Range)。

被拒绝的访问(目录不在白名单、路径穿越等)会明确报错,并记入主机端安全日志。

### 6.3 消息 / 安全审计 / 设置

- **消息**:与主机互发文字消息(双端持久化);
- **安全审计**:查看本会话相关的安全事件;
- **设置**:
  - 设备标签:保存后下次连接生效(连接页自动填入);
  - 主题(深色 / 浅色):点击即切换,刷新后保持;
  - 连接信息:查看会话 ID、主机名、系统;
  - 断开连接:断开并返回连接页(侧边栏底部的「断开连接」同理)。

### 6.4 会话被吊销时

主机端吊销你的会话后,页面会立即跳回连接页并提示「会话已被主机吊销」,需要主机重新生成 PIN 才能再次连接。

## 八、常见问题排查

| 现象 | 原因与处理 |
|---|---|
| Relay 启动报 `NODE_MODULE_VERSION ... requires ...` | better-sqlite3 二进制被 Electron 版覆盖 → `pnpm --filter @remotebridge/server rebuild better-sqlite3` |
| 桌面端显示"连接失败 / EACCES ::1:3001" | Relay 地址用了 `localhost` → 改为 `127.0.0.1`(设置页或配置文件) |
| 端口被占用(3000/3001) | `netstat -ano \| findstr :3001` 找到 PID 后结束进程,或换 `RELAY_PORT` |
| Web 输入 PIN 提示无效 | PIN 已过期(默认 5 分钟)或已被使用 → 主机端重新生成 |
| Web 文件页一直转圈 | 主机离线(看侧边栏状态);或目录刚被移出白名单 |
| 浏览器下载/预览失败且控制台有 CORS 错误 | Relay 的 `ALLOWED_ORIGINS` 未包含 Web 站点来源 → 补环境变量后重启 Relay |
| 桌面端窗口白屏 | 点击界面上的「重新加载」按钮(已内置错误兜底页) |
| 修改了 `packages/shared` 后行为异常 | 忘记重建 → `pnpm --filter @remotebridge/shared build` 后重启相关端 |

**数据存储位置**(备份/重置时参考):

| 数据 | 位置 |
|---|---|
| Relay 数据库(主机/会话/安全日志) | `~/.remotebridge/data/remotebridge.db`(可用 `RB_DATA_DIR` 改) |
| 桌面端配置(身份/Relay 地址/主题) | `AppData\Roaming\@remotebridge\desktop\remotebridge-config.json` |
| 桌面端本地库(白名单/客户端/消息/日志) | `AppData\Roaming\@remotebridge\desktop\data\local.db` |
| Web 端会话 | 浏览器 localStorage(清除站点数据即重置) |

## 九、快速上手清单(TL;DR)

```sh
# —— 终端 1:Relay ——
pnpm --filter @remotebridge/shared build
pnpm --filter @remotebridge/server dev

# —— 终端 2:Web ——
pnpm --filter @remotebridge/web dev

# —— 终端 3:桌面端 ——
pnpm --filter @remotebridge/desktop build
cd apps/desktop && .\node_modules\.bin\electron .
```

1. 桌面端左下角变绿(已连接);
2. 桌面端「共享目录」添加一个文件夹(选"可下载");
3. 桌面端「连接状态」生成连接码;
4. 浏览器打开 `http://localhost:3000`,输入连接码连接;
5. 「文件浏览」里点开目录 → 预览 / 下载文件。
