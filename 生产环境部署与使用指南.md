# RemoteBridge 生产环境部署与使用指南

> **适用版本**：v1.1.6+  
> **目标读者**：需要将 RemoteBridge 部署到公网服务器、通过域名访问 Web 端，并在人远离家庭/办公网络时仍能长期稳定连接桌面端的用户。

---

## 一、架构概述

```
互联网
  │
  ▼
[云服务器]
  ├── Caddy（TLS 终止，80/443 端口）
  ├── Relay Server（中继，仅监听内网 3001）
  └── Web Client（Next.js，仅监听内网 3000）

  ▲
  │ 桌面端主动向外发起 WebSocket 长连接（无需在家庭/公司网络开放任何端口）
  │
[家庭/办公室 PC]
  └── RemoteBridge 桌面端（Electron，outbound-only）
```

**核心优势**：桌面端始终是**主动向外连接** Relay Server，家庭路由器/防火墙不需要任何端口映射，NAT 穿透由此天然解决。只要云服务器可达，人在世界任何地方都能通过 Web 端访问桌面上的文件。

---

## 二、云服务器选型与准备

### 2.1 最低配置建议

| 项目 | 最低 | 推荐 |
|------|------|------|
| CPU | 1 核 | 2 核 |
| 内存 | 512 MB | 1 GB |
| 硬盘 | 10 GB SSD | 20 GB SSD |
| 带宽 | 1 Mbps | 5 Mbps+ |
| 系统 | Ubuntu 22.04 LTS | Ubuntu 24.04 LTS |

> 国内用户推荐轻量云服务器（腾讯云轻量/阿里云轻量/华为云）；海外用户推荐 Vultr、DigitalOcean、Hetzner。

### 2.2 安全组 / 防火墙规则

只需开放以下入站端口：

| 端口 | 协议 | 用途 |
|------|------|------|
| 80 | TCP | HTTP（Caddy 自动重定向到 HTTPS，Let's Encrypt 验证） |
| 443 | TCP | HTTPS / WSS（Caddy TLS 终止） |
| 22 | TCP | SSH 管理（按需限制来源 IP） |

**Relay Server（3001）和 Next.js（3000）绑定在内网，不对外暴露。**

### 2.3 安装 Docker 和 Docker Compose

```bash
# Ubuntu 22.04/24.04
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER   # 让当前用户免 sudo 使用 docker
newgrp docker                   # 立即生效（或重新登录）
docker compose version          # 确认 Compose v2
```

## 三、域名配置

### 3.1 购买域名并解析

1. 在任意域名注册商（阿里云万网、Namecheap、Cloudflare 等）购买域名，例如 `example.com`。
2. 在 DNS 管理面板添加 **A 记录**：

   | 主机记录 | 记录类型 | 记录值 |
   |----------|----------|--------|
   | `rb` | A | `<云服务器公网 IP>` |

   完整域名为 `rb.example.com`，根据喜好也可使用 `@`（裸域）或其他子域。

3. DNS 生效通常需要 5 分钟～1 小时，可用以下命令验证：

   ```bash
   nslookup rb.example.com
   # 应返回你的服务器 IP
   ```

> **如果使用 Cloudflare CDN**：先将代理模式设为「DNS only」（灰云），等 TLS 证书签好后再按需开启。

## 四、在服务器上部署

### 4.1 拉取代码

```bash
# 登录服务器（替换为实际 IP）
ssh user@<服务器IP>

# 克隆仓库
git clone https://github.com/Aswellle/RemoteBridge.git /opt/remotebridge
cd /opt/remotebridge
```

### 4.2 生成 JWT 密钥

```bash
# 生成两个独立的 48 字节随机密钥（必须各自独立，不得相同）
openssl rand -base64 48   # 第一个：用作 JWT_SECRET
openssl rand -base64 48   # 第二个：用作 JWT_REFRESH_SECRET
```

复制输出结果，稍后填入 `.env`。

### 4.3 创建 `.env` 配置文件

```bash
cp .env.example .env
nano .env   # 或 vim .env
```

将以下内容按实际情况填写：

```env
# ===== JWT 密钥（将 openssl 生成的值替换到这里）=====
JWT_SECRET=<第一个 openssl rand 输出>
JWT_REFRESH_SECRET=<第二个 openssl rand 输出>

# ===== 环境 =====
NODE_ENV=production

# ===== 域名（替换为你的实际子域名）=====
DOMAIN=rb.example.com

# ===== CORS：允许来自 Web 端域名的请求 =====
ALLOWED_ORIGINS=https://rb.example.com

# ===== Web 端访问 Relay 的地址（经 Caddy 代理，路径前缀 /api/v1 和 /ws）=====
NEXT_PUBLIC_API_URL=https://rb.example.com/api/v1
NEXT_PUBLIC_WS_URL=wss://rb.example.com/ws

# ===== 可选：限流（默认值通常够用）=====
RATE_LIMIT_MAX=10
RATE_LIMIT_WINDOW=60000
```

> `NEXT_PUBLIC_API_URL` 和 `NEXT_PUBLIC_WS_URL` 会在构建 Web 镜像时**内联到客户端 JS bundle**，修改后必须重新构建镜像才能生效（`docker compose build web`）。

### 4.4 首次构建并启动

```bash
cd /opt/remotebridge

# 构建三个服务的 Docker 镜像（首次较慢，约 5～15 分钟）
docker compose build

# 后台启动全部服务
docker compose up -d

# 查看启动状态
docker compose ps

# 查看 Relay 日志（确认无报错）
docker compose logs -f server
```

### 4.5 验证部署

```bash
# 健康检查（应返回 {"status":"ok",...}）
curl https://rb.example.com/health

# 在浏览器访问
# https://rb.example.com  → 应显示 RemoteBridge 连接页面
```

首次访问时 Caddy 会自动向 Let's Encrypt 申请 TLS 证书（约 10～30 秒），浏览器地址栏出现绿色锁即为成功。

## 五、桌面端配置（让家中 PC 连接到公网 Relay）

### 5.1 首次配置 Relay 地址

1. 打开 PC 上的 **RemoteBridge 桌面端**。
2. 进入「**设置**」→「**Relay 服务器**」，将默认的 `127.0.0.1:3002` 改为：

   | 字段 | 值 |
   |------|-----|
   | Relay WebSocket URL | `wss://rb.example.com/ws` |
   | Relay API URL | `https://rb.example.com/api/v1` |

3. 保存后桌面端会自动向公网 Relay 重新注册并建立长连接，托盘图标变绿即代表成功。

### 5.2 让桌面端开机自动运行

**Windows**：
- 打开「**任务计划程序**」→「创建基本任务」→ 触发器选「登录时」→ 操作选启动 `RemoteBridge.exe`。
- 或在「**启动**」文件夹（`Win+R` 输入 `shell:startup`）中放置快捷方式。

桌面端启动后会自动最小化到系统托盘，后台常驻，无需用户干预。

## 六、日常使用流程

### 6.1 Web 端连接桌面端

1. 在**桌面端**打开 RemoteBridge，点击「**生成连接码**」，得到 8 位 PIN 码（有效期 5 分钟）。
2. 在手机/平板/其他电脑浏览器访问 `https://rb.example.com`。
3. 在连接页面输入 PIN 码和本设备名称，点击「**连接**」。
4. 首次连接成功后，设备记录会保留在页面左侧历史面板，下次仍需生成新 PIN 重新认证（PIN 为一次性使用）。

### 6.2 文件操作

| 功能 | 操作 |
|------|------|
| 浏览文件 | 「文件」页面 → 点击共享目录进入 |
| 预览图片/文档/视频 | 点击文件名即触发预览（PDF 在浏览器原生阅读器打开） |
| 下载文件 | 点击文件 → 预览界面内点击下载，或目录列表右键下载 |
| 发送消息 | 「消息」页面输入后发送，桌面端消息中心实时收到 |

### 6.3 会话管理（桌面端）

- **信任设备**：在「已连接客户端」页面点击「信任」，将常用设备标记为可信任。
- **吊销会话**：设备丢失或换手时，点击「吊销」立即断开该设备的 WebSocket 连接，无法再用旧 token 访问。
- **查看活动日志**：「活动日志」标签可查看每次文件访问/下载的记录；「安全审计日志」记录认证失败、路径被阻止等安全事件。

## 七、保持长期稳定连接

### 7.1 桌面端自动重连机制

桌面端内置指数退避重连（1s → 2s → 4s → … → 最大 30s，**无限重试**）：

- 网络抖动、Relay 重启、PC 休眠唤醒后均会自动恢复连接，**无需任何手动操作**。
- 托盘图标灰色 = 正在重连；变绿 = 已重连成功。

### 7.2 防止 PC 休眠切断连接

长时间无人操作的 PC 若进入深度睡眠，网卡会断电，WebSocket 长连接随之中断（唤醒后会自动重连，但中间有空窗期）：

**Windows 设置防止睡眠**：
```
控制面板 → 电源选项 → 更改计划设置
  → "使计算机进入睡眠状态"：选「从不」
  → 显示器关闭时间：按需设置（不影响网络）
```

或使用命令行：
```powershell
# 永不休眠（仅屏幕保护）
powercfg /change standby-timeout-ac 0
powercfg /change standby-timeout-dc 0
```

### 7.3 Relay 服务器保活

Docker Compose 的 `restart: unless-stopped` 确保：
- Docker daemon 启动时自动拉起所有容器（等同开机自启）；
- 容器崩溃后 Docker 自动重启；
- 重启后两端自动重连，在途文件传输通过 HTTP Range 断点续传恢复。

在服务器上执行以下命令确认 Docker 开机自启：

```bash
sudo systemctl enable docker
```

### 7.4 外部健康监控（可选但推荐）

注册免费的 [Uptime Kuma](https://github.com/louislam/uptime-kuma)（可自建）或 [Healthchecks.io](https://healthchecks.io) 等服务，监控以下端点：

```
https://rb.example.com/health
```

当接口返回非 200 或 `"status":"error"` 时发送邮件/微信/Telegram 告警，第一时间感知 Relay 异常。

## 八、日常运维速查

### 查看服务状态

```bash
cd /opt/remotebridge
docker compose ps                  # 各服务运行状态
docker compose logs -f server      # Relay 实时日志
docker compose logs -f caddy       # Caddy 日志（证书/路由问题看这里）
```

### 重启服务

```bash
docker compose restart server      # 仅重启 Relay
docker compose restart             # 重启全部服务（含 Web 和 Caddy）
```

### 更新到新版本

```bash
cd /opt/remotebridge
git pull origin main               # 拉取最新代码
docker compose build               # 重新构建镜像（修改了 .env 中的 URL 也需此步）
docker compose up -d               # 滚动更新
docker compose ps                  # 确认健康
```

### 查看数据库大小与行数

```bash
curl -s https://rb.example.com/health | python3 -m json.tool
# 输出示例：
# {
#   "status": "ok",
#   "db": {
#     "sizeBytes": 1048576,
#     "tables": { "hosts": 3, "sessions": 28, "messages": 1540, "securityLogs": 220 }
#   }
# }
```

### 数据备份

SQLite 数据库存放于 Docker volume `remotebridge_server-data`，定期备份：

```bash
# 将数据库文件复制到宿主机
docker run --rm \
  -v remotebridge_server-data:/data \
  -v $(pwd)/backup:/backup \
  alpine cp /data/remotebridge.db /backup/remotebridge-$(date +%Y%m%d).db
```

## 九、常见问题排查

### Q1：浏览器访问域名显示「证书错误」

- 等待 30 秒后刷新，首次申请 Let's Encrypt 证书需要时间。
- 检查 80 端口是否开放：`docker compose logs caddy` 查找 `ACME` 或 `certificate` 关键字。
- 确认 DNS A 记录已生效：`nslookup rb.example.com` 应返回服务器 IP。

### Q2：桌面端托盘图标持续灰色，无法连接

1. 检查 Relay 是否正常：`curl https://rb.example.com/health`，应返回 `{"status":"ok"}`。
2. 检查桌面端设置里的 Relay 地址是否与部署域名一致（`wss://rb.example.com/ws`）。
3. 检查服务器防火墙是否放行了 443 端口（WSS 走 443，不是 3002）。
4. 若公司/校园网封锁了 WebSocket：使用手机热点测试；或联系网络管理员开放 443 WSS 流量。

### Q3：Web 端点击「连接」后提示「连接超时」或无响应

- 确认 PIN 码在 5 分钟内使用（过期需在桌面端重新生成）。
- 确认 Web 端的 `NEXT_PUBLIC_WS_URL` 与 Relay 域名一致（修改后需重新 `docker compose build web && docker compose up -d`）。
- 在浏览器开发者工具 Network 标签查看 WebSocket 握手是否成功（状态码应为 101）。

### Q4：文件下载速度慢

文件通过 Relay Server 中转（WS 文件隧道），速度受两段网络限制：
- **桌面端上行带宽**（家庭宽带通常为 10～50 Mbps 上行）。
- **Relay 服务器带宽**（轻量云服务器通常 1～5 Mbps）。

可升级云服务器套餐带宽，或选择国内 CDN 加速（需要额外配置）。

### Q5：长时间不用后 Web 端提示「会话已过期」

Access Token 有效期为 2 小时，Refresh Token 有效期为 30 天。Web 端会自动用 Refresh Token 续期，通常透明无感。若真正过期（30 天未使用）：

- 回到首页，在历史设备卡片提示下，让桌面端重新生成 PIN，完成重新认证即可。

## 十、安全建议

| 建议 | 说明 |
|------|------|
| 不定期更换 JWT 密钥 | 更换 `.env` 中的 `JWT_SECRET`/`JWT_REFRESH_SECRET` 后重启 server，所有现存 token 立即失效，用户需重新连接 |
| 定期检查已连接客户端 | 桌面端「已连接客户端」页面定期核查，对陌生设备立即吊销 |
| 启用安全审计日志告警 | Web 端「安全日志」频繁出现「认证失败」或「路径访问被阻止」时需警惕暴力破解 |
| 限制共享目录权限 | 只共享必要目录，敏感目录设为「只读」；系统目录（C:\Windows 等）已被自动拦截 |
| 服务器定期更新系统包 | `apt update && apt upgrade -y`，防止底层漏洞被利用 |
| 备份 SQLite 数据库 | 见第八节，建议每周自动备份到对象存储（OSS/S3） |
